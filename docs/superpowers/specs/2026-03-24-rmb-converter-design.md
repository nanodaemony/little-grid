# 人民币大写转换工具设计文档

## 概述

新增一个功能格子，提供人民币数字金额转大写汉字的功能，同时附带数字与人民币大写对照表，方便用户参考。

## 需求分析

### 功能需求
1. 输入数字金额，转换为对应的大写人民币汉字
2. 页面下方展示数字与人民币大写汉字的对照表
3. 界面设计需美观，符合应用整体风格

### 约束条件
- 支持金额范围：0 - 999,999,999.99 元（九亿九千九百九十九万九千九百九十九元九角九分）
- 支持小数点后两位（角、分）
- 遵循现有工具格子的设计模式

### 核心转换逻辑

#### 转换规则

1. **基本映射**
   - 0-9 对应：零、壹、贰、叁、肆、伍、陆、柒、捌、玖
   - 位数对应：拾、佰、仟、万、亿

2. **特殊处理**
   - 整数部分末尾加「整」字（无角分时）
   - 连续的零只保留一个「零」
   - 万位和亿位作为分隔点特殊处理
   - 10-19 之间的「壹拾」可简化为「拾」

3. **转换示例**
   ```
   1,234.56  → 壹仟贰佰叁拾肆元伍角陆分
   0.05      → 伍分
   10.00     → 壹拾元整
   100.50    → 壹佰元伍角
   10,001.00 → 壹万零壹元整
   10,000    → 壹万元整
   100,000,001.00 → 壹亿零壹元整
   ```

#### 算法概要

```
1. 将金额分为整数部分和小数部分
2. 处理小数部分（角、分）
   - 第一位：X角
   - 第二位：X分
   - 都为0：不加角分
3. 处理整数部分（从低位到高位分组）
   - 每4位为一组（个、十、百、千）
   - 组间添加单位（万、亿）
   - 处理组内零的情况
4. 合并结果，添加「元」和「整」
```

### 错误处理

| 场景 | 处理方式 |
|------|----------|
| 输入为空 | 显示提示：「请输入金额」 |
| 负数输入 | 显示提示：「金额不能为负数」 |
| 超出范围 | 显示提示：「金额过大，最大支持9亿9999万元」 |
| 格式错误 | 自动过滤，仅保留数字和小数点 |
| 多位小数 | 自动截断或四舍五入到两位 |

### 交互细节

1. **输入体验**
   - 输入时自动格式化（千分位逗号），使用自定义 `TextInputFormatter` 处理

   - 实现方式：使用 `TextEditingController` + `onChanged` 监听，或自定义 `NumberFormatter`

   - 实时验证输入合法性

### 视觉设计

1. **色彩方案**
   - 使用应用主色调
   - 结果区域使用浅色背景突出显示
   - 对照表使用卡片式布局

2. **字体层次**
   - 结果文字：大号字体（20-24sp），加粗
   - 输入文字：中等字号（16-18sp）
   - 对照表：常规字号（14sp）

3. **间距和圆角**
   - 卡片圆角：12-16dp
   - 模块间距：16-24dp
   - 内边距：16dp

## 接口定义

### RmbConverterService

```dart
class RmbConverterService {
  /// 将数字金额转换为人民币大写
  ///
  /// [amount]: 金额数字，如 1234.56
  /// 返回: 大写字符串，如 "壹仟贰佰叁拾肆元伍角陆分"
  static String convert(double amount);

  /// 验证金额是否在支持范围内
  static bool isValidAmount(double amount);

  /// 格式化输入（添加千分位）
  static String formatInput(String input);
}
```
