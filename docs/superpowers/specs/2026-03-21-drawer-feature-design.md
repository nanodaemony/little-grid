# 抽屉功能设计文档

## 概述
为"小方格"APP实现左侧抽屉导航功能，点击左上角菜单图标从左向右滑出。

## 设计目标
- 提供快捷访问设置、关于、反馈的入口
- 展示用户基本信息和使用统计
- 保持简洁，避免与底部导航功能重复

## 页面布局

### 整体结构
```
┌─────────────────────────┐
│  ╔═══════════════════╗  │
│  ║  [头像]           ║  │  ← Header（渐变背景）
│  ║  昵称 ▼           ║  │
│  ║  已使用 4 个工具   ║  │
│  ╚═══════════════════╝  │
│                         │
│  ⚙️ 设置        >       │
│                         │
│  ℹ️ 关于        >       │
│                         │
│  💬 反馈        >       │
│                         │
│         v1.0.0          │  ← Footer（版本号）
└─────────────────────────┘
```

### Header 区域
- **背景**：渐变（AppColors.primary → AppColors.primaryDark）
- **头像**：圆形，80px，白色背景，点击显示 TODO 提示（预留头像上传功能）
- **昵称**：20px 白色文字，点击弹出编辑对话框
- **统计**：显示"已使用 X 个工具"，14px 白色 70% 透明度

### 菜单项
| 图标 | 文字 | 点击行为 |
|------|------|----------|
| Icons.settings | 设置 | 关闭抽屉，跳转到 SettingsPage |
| Icons.info | 关于 | 关闭抽屉，弹出 AboutDialog |
| Icons.feedback | 反馈 | 关闭抽屉，显示 TODO SnackBar（功能预留）|

### Footer
- 居中显示版本号 "v1.0.0"（硬编码，后续可从 pubspec.yaml 读取）
- 12px，灰色文字

## 交互逻辑

### 打开抽屉
- 点击 GridPage AppBar 左上角的 menu 图标
- 调用 `Scaffold.of(context).openDrawer()`

### 点击菜单项
1. 关闭抽屉：`Navigator.pop(context)`
2. 导航到对应页面

### 编辑昵称
- 点击昵称区域
- 弹出 `AlertDialog` 输入框（与 ProfilePage 相同的对话框样式）
- 保存后更新本地状态（使用 StatefulWidget 的 setState）

### 编辑头像（预留）
- 点击头像
- 显示 SnackBar 提示"头像功能即将上线"

## 技术实现

### 修改文件
1. `lib/pages/grid_page.dart`
   - 添加 `drawer` 参数到 Scaffold
   - 实现 `_buildDrawer()` 方法

2. 新增 `lib/widgets/app_drawer.dart`（可选）
   - 将抽屉抽离为独立组件

### 数据结构
抽屉作为独立 StatefulWidget 组件，使用本地状态管理：
- `String _nickname` - 昵称，默认值为 '用户'
- 使用 `setState()` 更新状态
- 暂不需要持久化（与 ProfilePage 当前实现一致）

后续可考虑统一使用 `AppProvider` 或 `SharedPreferences` 实现跨页面状态同步。

### 尺寸规范
- 抽屉宽度：屏幕宽度的 75%
- Header 高度：约 180px
- 菜单项高度：56px
- 图标大小：24px
