# 忘记密码功能设计

**日期**: 2026-03-30
**目标**: 实现通过邮箱验证码重置密码的功能，支持后端和APP端

---

## 概述

为App用户添加忘记密码功能。用户输入手机号后，系统向绑定的邮箱发送验证码，用户通过验证码+新密码完成重置。同时支持用户绑定邮箱。

## API 接口设计

### 1. 绑定邮箱

**接口**: `POST /api/app/user/bind-email`

**请求体**:
```json
{
  "email": "user@example.com"
}
```

**响应**: `200 OK`

**错误码**:
- `400`: 邮箱格式错误
- `400`: 该邮箱已被其他用户绑定

**权限**: 需要登录（JWT Token）

---

### 2. 发送重置验证码

**接口**: `POST /api/app/auth/send-reset-code`

**请求体**:
```json
{
  "phone": "13800138000"
}
```

**响应**: `200 OK`

**错误码**:
- `400`: 手机号不存在
- `400`: 该手机号未绑定邮箱，请先绑定邮箱
- `400`: 验证码发送过于频繁（1分钟内只能发一次）

**逻辑**:
1. 验证手机号格式
2. 查询用户是否存在
3. 检查用户是否绑定了邮箱
4. 生成6位数字验证码
5. 存入 Redis，key 为 `reset:code:{phone}`，有效期 5 分钟
6. 调用邮件服务发送验证码

**邮件模板**:
```
您的验证码是：123456
验证码5分钟内有效，请勿泄露给他人。
```

---

### 3. 重置密码

**接口**: `POST /api/app/auth/reset-password`

**请求体**:
```json
{
  "phone": "13800138000",
  "code": "123456",
  "password": "加密后的新密码"
}
```

**响应**: `200 OK`

**错误码**:
- `400`: 验证码错误
- `400`: 验证码已过期
- `400`: 密码格式错误（需8位以上且包含字母和数字）
- `400`: 密码解密失败

**逻辑**:
1. 验证手机号格式
2. 从 Redis 获取验证码
3. 验证码匹配后删除 Redis 中的验证码
4. RSA 解密密码
5. 验证密码格式
6. 更新用户密码（BCrypt 加密）
7. 删除该用户的所有设备登录记录（强制重新登录）

## 服务层设计

### EmailVerifyService

**职责**: 处理密码重置的邮件验证码逻辑

**方法**:

```java
/**
 * 发送密码重置验证码
 * @param phone 手机号
 */
public void sendResetCode(String phone)

/**
 * 验证验证码并重置密码
 * @param phone 手机号
 * @param code 验证码
 * @param newPassword 新密码（已加密）
 */
public void resetPassword(String phone, String code, String newPassword)
```

**依赖**:
- `AppUserRepository`: 查询用户
- `RedisRepository`: 存储验证码
- `EmailService`: 发送邮件
- `AppUserDeviceRepository`: 清除登录记录

**Redis Key 设计**:
- 验证码: `reset:code:{phone}` (5分钟)
- 发送限制: `reset:limit:{phone}` (1分钟)

## 数据流图

### 忘记密码流程

```
用户: 输入手机号 → 点击"发送验证码"
  ↓
APP: POST /api/app/auth/send-reset-code
  ↓
后端: 查询用户 → 检查邮箱绑定 → 生成验证码 → 发送邮件 → 存Redis
  ↓
用户: 收到邮件 → 输入验证码 + 新密码
  ↓
APP: POST /api/app/auth/reset-password
  ↓
后端: 验证码 → 解密密码 → 更新密码 → 清除登录记录
  ↓
APP: 跳转回登录页
```

### 绑定邮箱流程

```
用户: 个人中心 → 点击"绑定邮箱"
  ↓
APP: 输入邮箱 → 绑定
  ↓
后端: 检查邮箱格式 → 检查是否已被占用 → 保存邮箱
  ↓
APP: 返回个人中心 → 显示已绑定的邮箱
```

## 安全设计

### 密码传输
- 前端使用 RSA 公钥加密密码
- 后端使用 RSA 私钥解密
- 密码使用 BCrypt 加密存储

### 验证码安全
- 验证码 5 分钟后自动失效
- 验证成功后立即删除 Redis 中的验证码
- 验证码只可使用一次
- 1 分钟内限制重复发送（防刷）

### 密码强度
- 最少 8 位
- 至少包含 1 个字母
- 至少包含 1 个数字
- 前端实时显示密码强度提示

### 重置后处理
- 重置成功后清除该用户的所有设备登录记录
- 强制用户重新登录，确保安全

## 错误处理

### 后端错误码

| 错误码 | 错误信息 | 场景 |
|:---|:---|:---|
| 400 | 手机号不存在 | 输入的手机号未注册 |
| 400 | 该手机号未绑定邮箱 | 用户未绑定邮箱，无法发送重置邮件 |
| 400 | 邮箱格式错误 | 绑定邮箱时格式不正确 |
| 400 | 该邮箱已被其他账号绑定 | 邮箱已被占用 |
| 400 | 验证码错误 | 输入的验证码不正确 |
| 400 | 验证码已过期 | 验证码超过 5 分钟 |
| 400 | 验证码发送过于频繁 | 1 分钟内重复发送 |
| 400 | 密码需8位以上且包含字母和数字 | 密码格式不符合要求 |
| 400 | 密码解密失败 | RSA 解密密码失败 |

### 前端错误处理

- 网络错误：显示"网络连接失败，请检查网络"
- 服务器错误：显示"服务器错误，请稍后重试"
- 业务错误：显示后端返回的错误信息
- 所有错误通过 SnackBar 展示
