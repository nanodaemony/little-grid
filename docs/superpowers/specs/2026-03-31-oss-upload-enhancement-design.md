# 阿里云 OSS 上传功能增强设计文档

**日期：** 2026-03-31
**模块：** eladmin-tools / eladmin-web
**状态：** 已批准

---

## 概述

在现有的 S3 对象存储页面中增强上传功能，支持多文件同时上传，上传完成后在上传对话框内展示每个文件的名称和对应的访问链接，每个链接旁提供复制按钮，方便用户快速获取文件链接并用于 APP 配置。

阿里云 OSS 完全兼容 S3 协议，因此无需后端改动，仅需修改前端页面即可实现。

---

## 需求背景

- APP 中需要大量图片资源
- 为了减小 APP 体积，图片存储在阿里云 OSS
- APP 中只存储图片链接，用户使用时按需下载
- 后台需要提供便捷的上传界面，上传后快速获取链接

## 后端接口

### 上传文件接口（复用）

**接口：** `POST /api/s3Storage`
**参数：** `MultipartFile file`
**返回格式：**
```json
{
  "id": 123,
  "errno": 0,
  "data": ["https://your-domain.com/path/to/file.jpg"]
}
```

**说明：**
- 无需修改后端代码
- 现有返回格式已包含所需的文件链接信息

## 前端实现

### 文件位置

`eladmin-web/src/views/tools/storage/s3/index.vue`

### 数据结构新增

```javascript
uploadResults: [
  {
    id: 123,
    fileName: 'image.jpg',
    fileUrl: 'https://domain.com/path/image.jpg',
    status: 'success', // 'success' | 'error'
    error: null // 错误信息（如有）
  }
]
```

### 新增功能点

1. **上传结果展示区域**
   - 在上传对话框底部添加分隔线
   - 渲染 `uploadResults` 列表
   - 每项显示：文件名、链接、复制按钮、状态图标

2. **复制链接功能**
   - 使用 Clipboard API 复制链接到剪贴板
   - 复制成功显示 Toast 提示
   - 复制失败提示手动复制

3. **成功回调处理**
   - 监听 `on-success` 事件
   - 解析后端返回的数据
   - 将结果添加到 `uploadResults` 数组

4. **错误处理**
   - 监听 `on-error` 事件
   - 记录失败文件信息
   - 显示错误状态

## 数据流

```
用户选择文件（支持多选）
    ↓
el-upload 组件触发上传
    ↓
调用 POST /api/s3Storage
    ↓
后端上传到阿里云 OSS
    ↓
后端返回 { id, errno, data: [url] }
    ↓
前端 on-success 回调
    ↓
解析文件名和链接
    ↓
将结果存入 uploadResults 数组
    ↓
渲染上传结果列表
    ↓
用户点击复制按钮
    ↓
调用 navigator.clipboard.writeText()
    ↓
复制成功，显示 Toast 提示
```

## 错误处理

### 上传失败
- 显示错误状态图标（❌）
- 显示错误信息（从后端响应获取）
- 不影响其他文件的上传

### 复制失败
- 浏览器不支持 Clipboard API 时，提示用户手动复制
- 提供文本框供用户选择复制

### 网络错误
- 显示重试按钮
- 保留已上传成功的文件信息

## 用户体验优化

1. **复制反馈**
   - 复制成功：显示 "链接已复制到剪贴板" Toast
   - 复制失败：显示 "复制失败，请手动复制"

2. **自动滚动**
   - 首个文件上传完成后，自动滚动到结果区域

3. **文件预览**
   - 点击文件名在新标签页打开链接
   - 方便验证上传是否成功

4. **状态指示**
   - 上传中：显示加载动画
   - 成功：绿色勾选图标
   - 失败：红色叉号图标
