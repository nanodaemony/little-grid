# 人生格子功能设计文档

## 概述

人生格子是一个时间可视化工具，通过小方格的形式展现时间流逝，让用户清晰感知时间的珍贵。

## 功能需求

### 核心功能
1. **多维度时间进度展示**：周进度、月进度、年进度、人生进度
2. **自定义进度**：用户可以创建自定义时间段（如学期、项目周期等）
3. **可视化呈现**：已过时间用蓝色格子表示，当前日期高亮显示
4. **用户设置**：支持调整显示哪些进度页、Tab顺序、人生进度参数

### 鼓励语
页面底部显示："一寸光阴一寸金"

### 主页面布局
1. **AppBar**: 标题"人生格子" + 右上角设置按钮（齿轮图标）
2. **Body**:
   - 如果用户隐藏了所有Tab，显示提示信息"请在设置中至少启用一个进度页"
   - 否则显示TabBar + TabBarView
3. **底部**: "一寸光阴一寸金" 鼓励词（灰色小字，居中）

### Tab顺序
- 默认顺序：周/月 → 年 → 人生 → 自定义
- 用户可在设置中调整Tab顺序
- 只显示用户启用的Tab

## 数据模型

### 用户设置 (LifeGridSettings)
```dart
class LifeGridSettings {
  // Tab显示控制
  bool showWeekMonth;    // 默认 true
  bool showYear;         // 默认 true
  bool showLife;         // 默认 true
  bool showCustom;       // 默认 true

  // Tab顺序
  List<String> tabOrder; // 默认 ['week_month', 'year', 'life', 'custom']

  // 人生进度设置
  DateTime birthDate;    // 出生日期
  int targetAge;         // 目标年龄，默认 80，范围 1-120

  // 当前激活的Tab索引
  int activeTabIndex;    // 默认 0
}
```

### 自定义进度 (CustomProgress)
```dart
class CustomProgress {
  String id;             // 唯一ID (UUID)
  String name;           // 名称，如"本学期"
  DateTime startDate;    // 开始日期
  DateTime endDate;      // 结束日期
  DateTime createdAt;    // 创建时间
}
```

### 存储方案
- **设置存储**: SharedPreferences，Key: `life_grid_settings`
- **自定义进度存储**: JSON序列化后存储，Key: `life_grid_custom_progresses`
- **数据迁移**: 新增字段时提供默认值

## 核心功能逻辑

### 周/月进度页面 (WeekMonthPage)

**周视图：**
- 显示本周7天，每天一个格子
- 7个格子排成一行
- 显示当前进度百分比

**月视图：**
- 显示本月所有天数，每天一个格子
- 每行7个格子（按周排列）
- 显示当前进度百分比

**切换方式：**
- 顶部有周/月切换按钮（SegmentedButton或ToggleButtons）

**计算逻辑：**
- 周进度：已过天数 / 7
- 月进度：已过天数 / 当月总天数

### 年进度页面 (YearPage)

**显示方式：**
- 显示全年365/366天
- 每月一组，每行显示7天（按周排列）
- 可垂直滚动查看全年
- 每月显示月份标签

**统计信息：**
- 当前是第几周
- 已过天数 / 总天数
- 已过百分比

### 人生进度页面 (LifePage)

**显示方式：**
- 显示 targetAge × 12 个格子
- 每行12个格子代表一年
- 每年左侧显示年龄数字（0岁、1岁...）
- 支持垂直滚动查看

**格子含义：**
- 每个格子代表一个月
- 已过去的月份显示蓝色
- 当前月份高亮显示（蓝色 + 金色边框）

**统计信息：**
- 已过人月数 / 总人月数
- 已过百分比
- 剩余月份数

**计算逻辑：**
- 已过人月数 = (当前日期年份 - 出生日期年份) × 12 + (当前日期月份 - 出生日期月份)
- 如果当前日期日期 < 出生日期日期，已过人月数减1

### 自定义进度页面 (CustomPage)

**列表视图：**
- 如果没有自定义进度，显示空状态（图标 + "点击添加自定义进度"）
- 列表显示所有自定义进度（名称 + 日期范围 + 进度百分比）
- 每个进度项点击进入详情页面
- 支持长按删除或滑动删除

**详情页面：**
- 显示该时间段的格子（每天一个格子）
- 格子大小类似周/月视图
- 支持编辑和删除

**添加流程：**
1. 点击"添加"按钮
2. 弹出对话框输入名称
3. 选择开始日期（日期选择器）
4. 选择结束日期（日期选择器）
5. 验证：结束日期必须晚于开始日期
6. 保存后自动进入详情页面

## 用户设置

### 设置入口
主页面右上角齿轮图标，点击后进入设置页面。

### 设置页面内容

#### 1. Tab显示设置
- 标题：显示哪些进度页
- 四个开关（Switch）：
  - [开关] 周/月进度
  - [开关] 年进度
  - [开关] 人生进度
  - [开关] 自定义进度
- 如果全部关闭，显示警告提示："至少选择一个进度页"

#### 2. Tab顺序设置
- 标题：调整页面顺序
- 可拖拽排序列表（使用 ReorderableListView）
- 只显示已启用的Tab
- 保存顺序到设置

#### 3. 人生进度设置
- 标题：人生进度参数
- [日期选择] 出生日期（使用日期选择器）
- [数字输入] 目标年龄（默认80，范围1-120）

#### 4. 数据管理
- [按钮] 重置所有设置（恢复默认值）
- [按钮] 清除所有自定义进度
- 操作前需要确认对话框

### 设置交互细节
- 设置变更立即生效（实时保存）
- 返回主页面时，如果当前显示的Tab被隐藏了，自动切换到第一个可用的Tab
- 使用本地存储，下次打开App保留设置

## 视觉设计

### 配色方案
- **已过去格子**：主色调蓝色（`AppColors.primary`）
- **未过去格子**：白色/透明，带浅灰色边框（`Colors.grey.shade300`）
- **当前日期格子**：蓝色填充 + 金色边框（`Colors.amber`）
- **背景色**：浅色背景（`AppColors.background`）

### 格子样式
- **形状**：圆角矩形（`BorderRadius.circular(4)`）
- **周/月进度**：格子约 40-50px，间距 4-6px
- **年进度**：格子约 20-25px，间距 2-3px
- **人生进度**：格子约 12-15px，间距 1-2px
- **自定义进度**：根据天数自适应，类似周/月大小

### 页面布局
- **顶部区域**：进度标题 + 统计信息（已过百分比、剩余天数等）
- **中部区域**：格子网格（可滚动）
- **底部区域**：鼓励词 "一寸光阴一寸金"（灰色小字，居中）

### TabBar 样式
- 使用 `TabBar`，固定在顶部（AppBar下方）
- 当前选中的Tab有下划线高亮
- 如果Tab较多，支持横向滚动
- Tab文字：周/月、年、人生、自定义

### 特殊标记
- 当前日期格子有金色边框标记
- 自定义进度的结束日期用浅红色标记
- 未来日期（未过去）显示为透明/白色带灰色边框

### 空状态
- 自定义进度页面无数据时，显示图标 + "点击添加自定义进度" + 添加按钮

### 统计信息展示
每个页面顶部显示：
- 当前时间位置（如"2026年3月第3周"）
- 进度百分比（大字体）
- 剩余时间（如"还剩5天"）

## 边界情况处理

### 人生进度边界
- 如果用户未设置出生日期，显示提示"请先在设置中设置出生日期"
- 目标年龄最小1岁，最大120岁
- 如果当前日期已经超过目标年龄，显示"已超出设定的人生目标年龄"

### 自定义进度边界
- 开始日期不能晚于结束日期
- 自定义进度名称不能为空
- 自定义进度数量无限制

### Tab显示边界
- 如果用户隐藏了所有Tab，显示提示信息并引导用户到设置
- Tab顺序列表只显示已启用的Tab

## 性能考虑

### 渲染优化
- 人生进度960个格子使用 `GridView.builder` 实现懒加载
- 使用 `const` 构造函数减少重建
- 格子状态计算缓存，避免重复计算

### 存储优化
- 设置变更防抖保存（300ms延迟）
- 自定义进度列表增量更新
